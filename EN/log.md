- [NEW] [[mopd]] — Created a paper-review page for MOPD (Multi-Domain On-Policy Distillation), the technique NVIDIA Nemotron-Cascade 2 uses (arXiv:2603.19220, Yang et al., NVIDIA, Mar 2026). Researched via one general-purpose agent that read the full 63-page tech report PDF. Findings that update what was previously assumed: **(1) MOPD is a SINGLE stage** inside the 7-stage Cascade RL pipeline (SFT → IF-RL → Multi-domain RL → **MOPD** → RLHF → Long-context RL → Code RL → SWE RL), NOT a per-round loop interleaved with each RL stage; the previous summary in [[on-policy-distillation]] was misleading and has been corrected. **(2) Only 3 teachers**, all FREE cascade by-products: math SFT checkpoint, RLHF side-branch (25 steps), Multi-domain RL best checkpoint — no extra training, no external models, no FP4 QAT or hidden-state caching needed (contrast with [[deepseek-v4-opd|DeepSeek-V4]]'s 10+ specialists with full-vocab KL). **(3) Naming collision**: the acronym MOPD was published 2 months earlier by Xiaomi MiMo-V2-Flash (arXiv:2601.02780, Jan 2026) as "Multi-**Teacher** On-Policy Distillation" — NVIDIA re-purposed the same letters as "Multi-**Domain**". Page captures the exact loss equations (Eq. 2-4 with sampled-token reverse-KL advantage, truncated importance weighting r_t ∈ [0.5, 2.0], stop-gradients), the cascade placement diagram, the headline experimental result (52 MOPD steps reach ArenaHard v2 85.5 vs RLHF's 160 steps for 80.7 — 3× per-step efficiency), the headline gold-medal results at 3B active parameters (IMO 35/42, IOI 439.28/600, ICPC 10/12 gold), the hyperparameter inconsistency between prose (LR 2e-6) and Table 8 (LR 3e-6), and where MOPD does NOT help (MMLU-Pro -5.5, GPQA -8.1, SWE Verified -19.0 vs Qwen3.5-35B-A3B — no GPQA / agentic teacher in the pool). Cross-linked at [[on-policy-distillation]] (corrected MOPD description) and [[deepseek-v4-opd]] (architectural contrast). EN and CN. Indexes updated.
