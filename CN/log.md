# 变更日志

## 2026-05-13
- [新增] [[af-disaggregation]] —— 给 Attention-FFN 分离单独建页，与 [[prefill-decode-disaggregation|PD 分离]] 并列。区分清楚：PD 沿时间轴切 *阶段*（prefill vs decode），AF 在单次 forward pass 内部切 *算子*（attention 层 vs FFN 层每层都跨网络）。页面覆盖：让 AF 值得做的不对称性（decode attention 是内存带宽瓶颈 ~1–2 FLOPs/byte、batched FFN 是算力瓶颈 1000+ FLOPs/byte；H100 roofline 拐点 ≈295 FLOPs/byte，统一硬件下任何时刻必有一类资源浪费）、架构（每层 hidden state 跨网络往返一次，带宽估算：Llama-70B 量级 batch 64 decode → 每 forward step ~160 MiB）、为什么 MoE 让 AF 自然（DP attention 与 EP MoE 的并行不兼容，跨算子 AllToAll 本来就在那）、具体系统（MegaScale-Infer = 异构硬件上的显式 AF；DeepSeek-V3 = 同硬件上的逻辑 AF（DP attention + EP MoE）；Mooncake = KV 存储解耦的邻近形态）、激活传输代价分析与平衡点（≥ 100 GPU、MoE active fraction ≤ 5%、NVLink 级互联）、PD 对比 AF 全表、PD + AF 叠加成 2×2 硬件矩阵的方法。EN/CN。
- [扩展] [[prefill-decode-disaggregation]] —— 新增"与 chunked prefill 的组合"一节，回答"PD 都把 prefill 拉到独立节点了为什么还要 chunked prefill" —— 三个 chunked prefill 在分离部署里仍然承重的场景（prefill 池内 prefill-prefill 干扰；decode 节点上的扩展 prefill 处理多轮 / 工具返回 / 投机解码回滚；池内流量整形）。带辨别两层的口诀。还加了一个"超越 PD：注意力-FFN 分离"指向新 [[af-disaggregation]] 页的简短小节。EN 加这两节；CN 原本就有 AF 短 stub，替换为指向 [[af-disaggregation]] 并补上 chunked-prefill-组合一节。
- [扩展] [[continuous-batching]] —— 在 Chunked Prefill 区域新增三个子节："为什么 prefill 会挡住 decode"（FLOPs 数学：Llama-70B × 16K = 2.3 PFLOPs ≈ H100 上 2.3 秒；解释一次 forward 是不可分的调度单元，切换粒度就是 iteration）、"Chunk size 的数学"（闭式推导：TBT = a·c+b、TTFT = a·T + b·T/c，所以小 chunk 反而让 TTFT 变差，是 b·T/c 这一项的固定开销税；甜点 512-2048 是 ∂TTFT/∂c 与边际 TBT 代价相等的点）、"FlashAttention 在 chunked prefill 里扮演什么角色"（FA 不改 attention FLOPs，只把内存峰值 O(S²)→O(S)；没有 FA 时 chunk size 被显存上限卡，被迫小；有 FA 后 chunk size 只受 TBT 偏好限制；FA-2/3 的 varlen-Q + paged-KV kernel 才是让 chunked prefill kernel 层面可行的真正原因）。EN/CN。
